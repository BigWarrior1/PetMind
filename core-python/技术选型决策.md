# 🎯 技术选型决策总结

## 一、硬件环境
- **显卡**: NVIDIA 3060 Laptop (6G显存)

- **开发优先级**: 快速开发 > 成本优化

## 二、最终技术选型

### ✅ 确定方案

| 组件 | 选型 | 理由 |
|------|------|------|
| **嵌入模型** | 阿里 text-embedding-v3 (API) | • 开发速度快,3行代码接入<br>• 与百炼平台无缝集成<br>• 节省显存,留给开发调试<br>• 前期调用量小,成本可控 |
| **向量数据库** | Chroma (本地) | • 与LangChain零配置集成<br>• Python原生实现<br>• 适合<10万文档规模<br>• 本地SQLite存储,无需部署 |
| **LLM模型** | 阿里百炼 qwen-plus | • 中文能力强<br>• API调用简单<br>• 成本适中 |
| **多模态模型** | 阿里百炼 qwen-vl-plus | • 图片理解能力好<br>• 与LLM同平台,统一管理 |
| **用户数据库** | SQLite → PostgreSQL | • 开发阶段用SQLite快速验证<br>• 上线时迁移到PostgreSQL |

## 三、为什么不选本地嵌入模型?

### 对比分析

| 维度 | 阿里API | 本地bge-large-zh-v1.5 |
|------|---------|----------------------|
| **开发速度** | ⚡ 5分钟接入 | ⏰ 需1-2天配置环境 |
| **显存占用** | ✅ 0GB (云端推理) | ❌ 约2GB (影响开发) |
| **代码复杂度** | ✅ 3行代码 | ❌ 需要模型加载/显存管理 |
| **前期成本** | ✅ <100元 (开发阶段) | ✅ 0元 |
| **后期成本** | ❌ 持续付费 | ✅ 免费 |
| **迁移难度** | ✅ 随时可切换本地 | - |

### 决策逻辑

```
毕业设计场景:
  时间紧 → 优先开发速度
  + 3060只有6G → 显存需留给调试/模型测试
  + 前期调用量小 → API成本可控
  = 选择阿里云API

上线后:
  调用量大 → 考虑成本
  + 项目稳定 → 有时间优化
  = 可迁移到本地bge模型
```
